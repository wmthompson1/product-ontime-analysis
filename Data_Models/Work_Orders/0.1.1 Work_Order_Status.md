**Visual Mfg help — status letter mapping (canonical)**
Source: `Visual Mfg Help Files/VM/Popup_Information_Cards.htm` and related help pages.

- `U` : Unreleased (On hold / not released)
- `F` : Firm (Firmed)
- `R` : Released
- `C` : Closed
- `X` : Cancelled

Notes:
- The Visual Mfg help shows the single-character codes used in UI popups and dialogs; treat this mapping as the canonical user-facing definitions. Some repository comments or queries may use different wording (for example some comments label `F` as "Finished"). Prefer "Firm" for `F` in documentation.

**Recommended conventions**
- Use the Visual Mfg mapping as authoritative when naming statuses in docs and UI-facing messages: `F = Firm` (not "Finished").
- For operational queries that should exclude inactive orders, use: `WHERE WO.STATUS NOT IN ('C','X')`.
- For planning/production selection, use: `WHERE WO.STATUS IN ('R','F')` to capture released + firmed orders.
- Avoid relying on textual comments that rename letters (e.g., "Finished") — prefer `Firm/Released/Closed/Cancelled` in comments to match help text.
