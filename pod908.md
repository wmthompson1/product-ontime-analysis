# 📜 **Part II — Section 4: SQLMesh Enforcement Controls**  
## **4.1 Masking Matrix + Certificate Two‑Key Rule**

### **Rule 1 — Masking Matrix Defines Policy**
All masking rules must be declared in `masking_matrix.csv`.

### **Rule 2 — Certificate Defines Attestation**
Before ingestion, the receiving agent must:

- Increment `certificate_version`
- Update `masking_matrix_ref`
- Update `dag_lines` to match the matrix
- Set `status = certified`

### **Rule 3 — SQLMesh Will Not Ingest Without a Valid Certificate**
SQLMesh checks:

- The certificate version  
- The DAG lines  
- The trust boundary  
- The receiving agent ID  

If the certificate does not match the matrix:

- SQLMesh halts ingestion  
- No stg.* models are refreshed  
- No hashed models are rebuilt  
- No backfills are triggered  

### **Rule 4 — Backfills Require Certificate Alignment**
If the matrix changes:

- SQLMesh computes a plan  
- SQLMesh schedules backfills  
- SQLMesh rebuilds downstream models  
- SQLMesh promotes only after certificate validation
