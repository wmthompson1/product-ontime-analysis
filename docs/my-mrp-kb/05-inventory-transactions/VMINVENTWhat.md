# What is Inventory Transaction Entry?

Use Inventory Transaction Entry to perform these functions:

* [Issuing](VMINVENTfrmIssue.md)
  and [returning](Returning_Issued_Materials.md) inventory
  materials to work orders. Materials purchased directly to work
  orders are issued by Purchase Receipt Entry, and cannot be issued
  from here. Use the main Inventory Transaction Entry window to
  issue materials one-by-one. To issue all materials required by
  a work order, use the [Issue
  by Exception](VMINVENT_APLfrmIssue.md) function.
* [Receiving](Receiving_Materials_Into_Inventory.md)
  and [returning](Returning_Received_Materials.md) finished
  goods from work in process. Work orders fully linked to a customer
  order are received to inventory and issued to the customer order
  automatically by Shipping Entry, and cannot be received from here.
  You can receive finished goods by work order or [by
  part](Receiving_by_Part.md).
* [Adjusting
  inventory](Adjusting_Materials.md) on-hand quantities to reconcile with actual physical
  quantities. Physical Inventory Count does this automatically.
* [Viewing](Viewing_and_Modifying_Transaction_Information.md)
  of inventory transactions created automatically by Purchase Receipt
  Entry, Shipping Entry, Physical Inventory Count, and Labor Ticket
  Entry auto-issues.
* [Transferring
  inventory](VMINVENT_APLfrmInventoryTransfer.md) between part locations
